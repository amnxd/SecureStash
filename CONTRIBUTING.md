## How to contribute

1. Fork the repository and create a branch for your change:

    git checkout -b feature/your-feature

2. Make small, focused changes and include tests when possible.

3. Run the project's tests and linters before opening a PR:

    npm install
    npm test
    npm run lint

4. Push your branch and open a Pull Request against `main` with a clear description of the change.

5. Be responsive to review comments and update your PR accordingly.

## Code style and testing

- The project uses TypeScript and React. Keep code readable and typed where appropriate.
- Add unit tests for new features and bug fixes. We use Jest for unit tests.

## Reporting security issues

If you find a security vulnerability, please open a private issue or contact the repository owner rather than posting it publicly.
